# 세탁 브랜드 AS 관리 페이지 — 설계 스펙

- 작성일: 2026-07-22
- 배포 방식: GitHub (새 저장소, GitHub Pages 정적 배포)
- 백엔드: Google Apps Script 웹앱 (Google Sheets를 데이터 저장소로 사용)

## 1. 배경 및 목적

세탁 회사가 브랜드 AS(백화점 매장 방문 접수)를 관리하는 업무를 현재 슬랙 워크플로우 + 구글 스프레드시트로 처리하고 있음. 이를 대체할 전용 관리 웹앱을 만든다.

- 접수자(로그인한 직원)가 AS 건을 입력
- 관리자가 목록/대시보드로 전체 현황 파악, 상태 관리
- 매장을 실제로 방문하는 현장 담당자가 모바일에서 간단히 진행 상황을 갱신

## 2. 프로젝트 위치 및 배포

- 완전히 새 폴더 / 새 GitHub 저장소로 시작
- 정적 프론트엔드는 GitHub Pages로 배포
- 백엔드 로직(데이터 CRUD, 인증 검증, 집계)은 Google Apps Script 웹앱으로 배포하고, 프론트에서 `fetch`로 호출

## 3. 인증

- 로그인: Google Identity Services(GIS) 버튼 → ID 토큰 발급
- 프론트가 ID 토큰을 Apps Script `action=login`으로 전송
- Apps Script가 Google `tokeninfo` 엔드포인트로 토큰 검증 → 이메일 추출
- `직원목록` 시트에서 화이트리스트 + 활성여부 + 역할(일반/관리자) 조회
  - 화이트리스트에 없거나 비활성 상태면 로그인 거부
- 통과 시 `{이메일, 이름, 역할}`을 응답하고, 프론트는 localStorage에 세션 저장(8시간 만료)
- 화이트리스트(직원 이메일 추가/삭제/활성화/역할 변경)는 **관리자 역할**만 설정 탭에서 관리 가능

## 5. 화면 구성 (탭)

단일 페이지 안에서 탭 전환: **대시보드 / 접수 / 목록 / 현장 / 설정**

### 5.1 대시보드 탭

- 접수 필요 / 회수 필요 건수 카드
- 경과기간별 건수: 2주 이하 / 3주 이상 / 4주 이상 / 5주 이상 / 5주 초과 (수거요청일자와 오늘 날짜 차이로 서버에서 계산)
- 담당자별(로그인 이메일 → `직원목록` 이름 매칭 기준) 진행중 건수 집계
- Apps Script `action=dashboard` 하나로 집계 결과만 응답 (Raw Data 전체를 프론트로 내려주지 않음)

### 5.2 접수 탭 (폼)

- 4.1의 필드를 그대로 폼으로 구현
- 고객분류 = "런드리고"를 선택했을 때만 "배송 완료 처리" 드롭다운 노출
- 제출 시 `action=submitAS` 호출 → 서버가 로그인 이메일로 `직원목록`에서 이름을 조회해 `접수자` 값으로 채운 뒤 `AS접수` 시트에 새 행 추가 (상태=기본값 "접수 필요")
- 필수 필드 누락 시 제출 전 검증, 성공/실패 토스트 표시

### 5.3 목록 탭

- `action=listAS`로 전체 건 조회, 최신순 표시
- 상단 필터 칩: 전체 / 접수 필요 / 회수 필요 / 중동사 진행 / 아토즈레더 진행 / 택배 접수
- 경과기간별 정렬/필터(2주 이하 ~ 5주 초과) 옵션 제공
- 카드/테이블에서 핵심 정보(브랜드, 품목, 고객분류, 상태, 접수일) 노출, 클릭 시 상세 필드 펼침
- 상태 드롭다운으로 즉시 변경 → `action=updateStatus` 호출

### 5.4 현장 탭 (신규, 모바일 반응형)

- 매장을 실제 방문하는 담당자가 사용. 로그인한 사용자 기준으로 항목 목록을 카드로 표시 (모바일 화면에서 보기 편한 반응형 레이아웃)
- 각 카드에 간단 상태 버튼 3개: **AS불가 / 진행중 / 수거완료**
  - 선택 시 상세 상태값으로 자동 매핑 후 저장:
    - AS불가 → `AS 불가`
    - 진행중 → `AS 진행중`
    - 수거완료 → `회수 완료`
  - 매핑 관계는 프론트/서버 코드에 고정값으로 둠 (설정 탭에서 상태값 자체는 추가/삭제 가능하지만, 이 3개 매핑은 이번 스코프에서는 하드코딩)
- 메모 입력란(자유 텍스트, `현장메모` 컬럼에 저장)
- `action=fieldUpdate` 호출 (상태 + 메모 동시 저장)

### 5.5 설정 탭 (관리자 전용)

- 역할이 "관리자"인 사용자에게만 탭 노출 (프론트에서 숨김 + 서버에서도 역할 재검증)
- **직원관리**: 이메일 추가/삭제, 활성화 토글, 역할 변경 (`action=listStaff / addStaff / updateStaff`)
- **상태값관리**: 상태값 추가/삭제 (`action=listStatus / addStatus / deleteStatus`)

## 6. Apps Script API 액션 목록

| action | 설명 | 권한 |
|---|---|---|
| login | ID 토큰 검증 + 화이트리스트 조회 | 전체 |
| submitAS | AS 접수 건 생성 | 로그인 사용자 |
| listAS | AS 접수 목록 조회 | 로그인 사용자 |
| updateStatus | 상태값 변경 | 로그인 사용자 |
| deleteAS | AS 접수 건 삭제 | 관리자 |
| fieldUpdate | 현장 간단 상태 + 메모 저장 | 로그인 사용자 |
| dashboard | 집계 데이터 조회 | 로그인 사용자 |
| listStaff / addStaff / updateStaff / deleteStaff | 직원 화이트리스트 관리 | 관리자 |
| listStatus / addStatus / deleteStatus | 상태값 목록 관리 | 관리자 |

## 7. 에러 처리

- Apps Script 응답 실패, 네트워크 오류 시 한글 에러 메시지 + 재시도 버튼 표시
- 관리자 전용 액션은 서버 측에서도 역할 재검증 (프론트 숨김만으로 끝내지 않음)

## 8. 스코프 밖 (1차 제외)

- 사진/이미지 첨부
- 바코드 카메라 스캔 (텍스트 입력만)
- 알림(슬랙/이메일) 연동
- 상세 상태값 ↔ 현장 3단계 매핑의 관리자 커스터마이징 (현재는 하드코딩)
